## Usage

Use `ComposedPopover` for the convenience API: pass a `trigger` and the body; Content (portal + positioner + arrow) is auto-injected.

```tsx
import { Button, ComposedPopover, Popover } from '@gravitational/design-system';

function Example() {
  return (
    <ComposedPopover
      placement="bottom-start"
      trigger={<Button>Resource tips</Button>}
    >
      <Popover.Body>
        <Popover.Title fontWeight="bold">Finding resources</Popover.Title>
        <div>Filter by labels, names, or kinds.</div>
      </Popover.Body>
    </ComposedPopover>
  );
}
```

### Placements

Use the `placement` prop to position the popover relative to its trigger. Accepts any Chakra placement value (`top`, `top-start`, `top-end`, `right`, …).

### Compose mode

For full compositional control, use `Popover.Root` with the slot parts directly. This skips the auto-injection in `ComposedPopover`, giving you control over the positioner, arrow, and any custom header/footer structure.

```tsx
import { Button, CloseButton, Popover } from '@gravitational/design-system';

function Example() {
  return (
    <Popover.Root positioning={{ placement: 'bottom-start' }}>
      <Popover.Trigger asChild>
        <Button>Session recording</Button>
      </Popover.Trigger>
      <Popover.Positioner>
        <Popover.Content>
          <Popover.Arrow>
            <Popover.ArrowTip />
          </Popover.Arrow>
          <Popover.CloseTrigger asChild>
            <CloseButton size="sm" />
          </Popover.CloseTrigger>
          <Popover.Header>Session recording</Popover.Header>
          <Popover.Body>Body content here.</Popover.Body>
          <Popover.Footer>Last sync: 2m ago</Popover.Footer>
        </Popover.Content>
      </Popover.Positioner>
    </Popover.Root>
  );
}
```

### With a form

Popovers are a natural fit for quick-edit UIs. Put fields in `Popover.Body` and actions in `Popover.Footer`. Widen the content with `contentProps={{ minW: '...' }}` if the default is too tight.

<Canvas of={PopoverStories.WithForm} />

### Initial focus

By default the first focusable element in the popover receives focus on open. Pass `initialFocusEl` with a ref to target a specific element - useful for tours where the primary action is halfway down the content.

<Canvas of={PopoverStories.InitialFocusPopover} />

### Nested popovers

Popovers can be nested. The parent stays open, the child stacks on top, and focus returns to the child's trigger on dismiss.

<Canvas of={PopoverStories.NestedPopover} />

### Controlled with `anchorRef`

Drive the popover programmatically by supplying `open` / `onOpenChange` plus an `anchorRef`. No trigger is rendered - the popover positions itself against the referenced element.

<Canvas of={PopoverStories.ControlledPopover} />

### Layout

Compose `Popover.Header`, `Popover.Body`, `Popover.Footer`, and `Popover.CloseTrigger` for richer content.

<Canvas of={PopoverStories.Layout} />

### Positioning

Use `gutter` for a simple distance-from-anchor, or the full `positioning` config for fine-grained control (offset, flip, overflow padding, …). `positioning` spreads last and wins over flat shortcuts.

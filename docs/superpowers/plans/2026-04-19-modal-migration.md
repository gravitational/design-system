# Modal Migration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers-extended-cc:subagent-driven-development (recommended) or superpowers-extended-cc:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship a `Modal` component in `@gravitational/design-system` that wraps Chakra v3's `Dialog.*` primitives, preserves Teleport's `Dialog` API surface (onClose with reason, hideBackdrop, keepInDOMAfterClose, disable\*, trapFocus, etc.), and follows Chakra's defaults.

**Architecture:** Thin wrapper around Chakra v3's `Dialog.*` backed by a slot recipe registered under the Chakra key `dialog` (matching `dialogAnatomy`). The wrapper supports compositional + `trigger=` convenience modes, synthesizes Teleport's `onClose(event, reason)` callback by intercepting Ark's `onEscapeKeyDown` and `onInteractOutside`, and attaches all Chakra sub-components as static properties. No `anchorRef` mode (modals are centered, not anchored).

**Tech Stack:** TypeScript, React 19, `@chakra-ui/react@^3.31.0`, `@ark-ui/react`, Storybook (Vite), `vitest`.

**Spec:** `docs/superpowers/specs/2026-04-19-modal-migration-design.md`

```tsx
import { Dialog as ChakraDialog, Portal } from '@chakra-ui/react';
import type { ReactNode, RefAttributes } from 'react';

type CloseReason = 'escapeKeyDown' | 'backdropClick';

export interface ModalProps extends Omit<ChakraDialog.RootProps, 'children'> {
  /**
   * Fired when the component requests to be closed via Escape or an
   * outside interaction. Close-button clicks go through `onOpenChange`.
   */
  onClose?: (
    event: KeyboardEvent | PointerEvent | FocusEvent,
    reason: CloseReason
  ) => void;
  /** Fires on backdrop / outside interaction, before `onClose`. */
  onBackdropClick?: (event: PointerEvent | FocusEvent) => void;
  /** Fires on Escape keydown, before `onClose`. */
  onEscapeKeyDown?: (event: KeyboardEvent) => void;

  /** If `true`, clicking the backdrop does NOT close the modal. */
  disableBackdropClick?: boolean;
  /** If `true`, Escape does NOT close the modal. */
  disableEscapeKeyDown?: boolean;
  /** If `true`, focus is not restored after close. */
  disableRestoreFocus?: boolean;

  /** If `true`, the backdrop is not rendered. Convenience mode only. */
  hideBackdrop?: boolean;
  /** If `true`, the modal remains mounted after close. */
  keepInDOMAfterClose?: boolean;

  /** Props forwarded to the auto-injected `<Modal.Backdrop>`. */
  backdropProps?: ChakraDialog.BackdropProps;
  /** Props forwarded to the auto-injected `<Modal.Content>`. */
  contentProps?: ChakraDialog.ContentProps;

  /**
   * Convenience: element wrapped in `<ChakraDialog.Trigger asChild>`.
   * When set, the wrapper auto-injects Backdrop + Positioner + Content.
   */
  trigger?: ReactNode;

  children?: ReactNode;
}

export function Modal({
  onClose,
  onBackdropClick,
  onEscapeKeyDown,
  disableBackdropClick,
  disableEscapeKeyDown,
  disableRestoreFocus,
  hideBackdrop,
  keepInDOMAfterClose,
  backdropProps,
  contentProps,
  trigger,
  children,
  ref,
  ...rest
}: ModalProps & RefAttributes<HTMLDivElement>) {
  const rootProps = {
    ...rest,
    closeOnEscape: !disableEscapeKeyDown,
    closeOnInteractOutside: !disableBackdropClick,
    restoreFocus: !disableRestoreFocus,
    unmountOnExit: !keepInDOMAfterClose,
    onEscapeKeyDown: (event: KeyboardEvent) => {
      onEscapeKeyDown?.(event);
      if (!disableEscapeKeyDown) onClose?.(event, 'escapeKeyDown');
    },
    onInteractOutside: (
      event: CustomEvent<{ originalEvent: PointerEvent | FocusEvent }>
    ) => {
      const original = event.detail.originalEvent;
      onBackdropClick?.(original);
      if (!disableBackdropClick) onClose?.(original, 'backdropClick');
    },
  };

  if (trigger === undefined) {
    return <ChakraDialog.Root {...rootProps}>{children}</ChakraDialog.Root>;
  }

  return (
    <ChakraDialog.Root {...rootProps}>
      <ChakraDialog.Trigger asChild>{trigger}</ChakraDialog.Trigger>
      <Portal>
        {!hideBackdrop && <ChakraDialog.Backdrop {...backdropProps} />}
        <ChakraDialog.Positioner>
          <ChakraDialog.Content ref={ref} {...contentProps}>
            {children}
          </ChakraDialog.Content>
        </ChakraDialog.Positioner>
      </Portal>
    </ChakraDialog.Root>
  );
}

Modal.Trigger = ChakraDialog.Trigger;
Modal.Backdrop = ChakraDialog.Backdrop;
Modal.Positioner = ChakraDialog.Positioner;
Modal.Content = ChakraDialog.Content;
Modal.Header = ChakraDialog.Header;
Modal.Body = ChakraDialog.Body;
Modal.Footer = ChakraDialog.Footer;
Modal.Title = ChakraDialog.Title;
Modal.Description = ChakraDialog.Description;
Modal.CloseTrigger = ChakraDialog.CloseTrigger;
Modal.ActionTrigger = ChakraDialog.ActionTrigger;
```

Notes:

- Ark's `onInteractOutside` receives a `CustomEvent<EventDetails<PointerEvent | FocusEvent>>`. We unwrap `event.detail.originalEvent` before forwarding to callers so the API matches Teleport's DOM-event style. If typegen disagrees about the event shape, fall back to `(event as any).detail.originalEvent` — the runtime behavior is stable.
- In compositional mode, all custom props unrelated to `ChakraDialog.RootProps` (like `hideBackdrop`) are silently ignored. The JSDoc already documents this.

## Usage

Compose the modal from a trigger, backdrop, positioner, and content.

```tsx
import { Button, Modal } from '@gravitational/design-system';

function Example() {
  return (
    <Modal>
      <Modal.Trigger asChild>
        <Button>Open</Button>
      </Modal.Trigger>
      <Modal.Backdrop />
      <Modal.Positioner>
        <Modal.Content>
          <Modal.Header>
            <Modal.Title>Title</Modal.Title>
          </Modal.Header>
          <Modal.Body>Content here.</Modal.Body>
          <Modal.Footer>
            <Modal.CloseTrigger asChild>
              <Button>Close</Button>
            </Modal.CloseTrigger>
          </Modal.Footer>
        </Modal.Content>
      </Modal.Positioner>
    </Modal>
  );
}
```
````

### Motion presets

Use `motionPreset` to choose an open/close animation.

<Canvas of={ModalStories.MotionPreset} />

### Hide backdrop

Set `hideBackdrop` to render without a dimmed overlay. Clicks behind the modal pass through.

<Canvas of={ModalStories.HideBackdrop} />

### Keep in DOM after close

Set `keepInDOMAfterClose` to preserve the modal's subtree (and any internal state) across close cycles.

<Canvas of={ModalStories.KeepInDOMAfterClose} />

### Disable close affordances

Pair `disableBackdropClick` and `disableEscapeKeyDown` for destructive confirmations where accidental dismissal is unacceptable.

<Canvas of={ModalStories.DisableClose} />

### Close reason

`onClose(event, reason)` fires for Escape and outside-click dismissals. Close-button clicks go through `onOpenChange` instead.

<Canvas of={ModalStories.CloseReason} />
